Hash Table Benchmarks (2013)¶

Recently, I found Nick Welch’s Hash Table Benchmark, in which he compare the most noteable C/C++ hash table implementations. It was curious to know performance of different implementations of the hash tables with recent librarys versions, and see the changes after 4 years. Of course the main interest is Google hashs tables.

Unfortunately, I have not so powerfull notebook with only 3Gb RAM for test, by the time of writing of this note, so I had to change some test’s vars, to fit my hardware. I removed out of the test Ruby and Qhash, because I wanted a more of clear graphs.

Sequential Inserts: Execution Time (integers)

Original Nick’s benchmark with values up to 16 000 000.

The main surprise caused performance of the Boosts. Unfortunately due to the fact that there was no time to check out other versions of the Boost, it is not clear whether this is a result of regression in the performance of the current version.

Also I want to note that with Random strings inserts operations, which consume more memory, with Python test, I ran out of memory at 14M, that cause using SWAP up to 100Mb.

It is clearly seen that Google’s sparse_hash_map is significantly accelerated, while memory consumption not changed.
